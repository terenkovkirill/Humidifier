Самодельный **ультразвуковой увлажнитель воздуха** на базе **ESP8266** с управлением через **Telegram-бота**.  
Прошивка измеряет температуру и влажность, автоматически поддерживает заданный уровень влажности и отправляет уведомления о низком уровне воды.

---

## О проекте

Этот репозиторий содержит прошивку для IoT-увлажнителя воздуха, собранного на базе ESP8266.  
Идея проекта выросла из учебного предложения по созданию автономного ультразвукового увлажнителя с удалённым управлением, датчиком температуры и влажности, а также контролем уровня воды.

В текущей реализации устройство:
- считывает температуру и влажность с **DHT22**;
- управляет увлажнителем через цифровой выход;
- поддерживает **автоматический режим** по целевой влажности;
- принимает команды из **Telegram**;
- отслеживает уровень воды по аналоговому датчику;
- отправляет уведомления при изменении состояния уровня воды.

---

## Возможности

- **Автоматическое поддержание влажности** по заданной уставке
- **Ручное управление** через Telegram-команды
- **Три режима работы**: `AUTO`, `FORCE ON`, `FORCE OFF`
- **Мониторинг температуры и влажности** с DHT22
- **Контроль уровня воды** через аналоговый вход `A0`
- **Уведомления в Telegram** при низком уровне воды
- **Поддержка нескольких Wi‑Fi сетей** через `ESP8266WiFiMulti`
- **Автопереподключение к сети** при обрыве связи

---

## Как работает логика управления

В прошивке реализованы три режима:

- `MODE_AUTO` — автоматическое поддержание влажности
- `MODE_FORCE_ON` — принудительное включение увлажнителя
- `MODE_FORCE_OFF` — принудительное выключение

В автоматическом режиме используется **гистерезис 2%**:
- если влажность опускается ниже `targetHum - 2`, увлажнитель включается;
- если влажность поднимается выше `targetHum + 2`, увлажнитель выключается.

Это уменьшает частые переключения и делает работу стабильнее.

## Важное замечание по безопасности

> [!WARNING]
> В текущей версии прошивки **защита от работы без воды отключена**.
> 
> Датчик воды используется **только для статуса и уведомлений**. Если воды мало, бот пришлёт предупреждение, но устройство **не выключится автоматически**.
> 
> Не рекомендуется оставлять устройство без присмотра, пока не добавлена аппаратная или программная защита по уровню воды.

## Аппаратная часть

### Используемые компоненты

- **ESP8266** (NodeMCU / Wemos D1 mini)
- **DHT22**
- **Ультразвуковой модуль увлажнителя**
- **MOSFET / силовой ключ** для управления модулем
- **Аналоговый датчик уровня воды**
- Источник питания

### Пины в текущем скетче

| Назначение | Пин платы | GPIO |
|---|---:|---:|
| DHT22 | `D2` | `GPIO4` |
| Управление увлажнителем | `D5` | `GPIO14` |
| Датчик уровня воды | `A0` | Analog |

```cpp
#define DHTPIN D2
#define DHTTYPE DHT22

const int HUMIDIFIER_PIN   = D5;
const int WATER_SENSOR_PIN = A0;
```

## Команды Telegram-бота

| Команда | Описание |
|---|---|
| `/start` | Показать список команд |
| `/on` | Принудительно включить увлажнитель |
| `/off` | Принудительно выключить увлажнитель |
| `/status` | Отправить текущий статус устройства |
| `/setXX` | Установить целевую влажность, например `/set50` |

### Что показывает `/status`

Команда возвращает:
- текущую температуру;
- текущую влажность;
- целевую влажность;
- активный режим работы;
- состояние увлажнителя;
- состояние датчика воды.

## Настройка перед прошивкой

### 1. Укажите Wi‑Fi сети

```cpp
wifiMulti.addAP("YOUR_WIFI_1", "YOUR_PASSWORD_1");
wifiMulti.addAP("YOUR_WIFI_2", "YOUR_PASSWORD_2");
```

### 2. Укажите токен Telegram-бота

```cpp
#define BOT_TOKEN "YOUR_BOT_TOKEN"
```

### 3. При необходимости откалибруйте порог датчика воды

```cpp
const int WATER_DRY_THRESHOLD = 500;
```

Практически лучше сначала вывести сырые показания датчика воды в `Serial` и подобрать значение порога под конкретную конструкцию бака.

---

## Сборка и загрузка прошивки

### Через Arduino IDE

1. Установите поддержку плат **ESP8266**.
2. Установите необходимые библиотеки.
3. Откройте скетч.
4. Укажите Wi‑Fi и `BOT_TOKEN`.
5. Выберите свою плату (`NodeMCU` / `Wemos D1 mini`).
6. Загрузите прошивку на устройство.
7. Откройте `Serial Monitor` на скорости **115200 бод**.

## Особенности текущей реализации

- используется `WiFiClientSecure` с `setInsecure()` для упрощённой работы Telegram на ESP8266;
- прошивка хранит `lastChatId`, чтобы отправлять уведомления в последний активный чат;
- поддерживается работа сразу с несколькими Wi‑Fi сетями;
- логика достаточно простая и подходит как для учебного, так и для домашнего DIY-проекта.

## Примечание

В исходном проектном описании идея формулировалась как ультразвуковой увлажнитель с автономной работой, удалённым управлением, DHT22 и Telegram-ботом.
